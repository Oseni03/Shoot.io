## Context

The extension (WXT + React 19 + Tailwind v4) has a working scaffold with login flow from [001](./001-extension-scaffold-login.md). The service worker (`background.ts`) handles `LOGIN`, `GET_ME`, `LOGOUT` via `AuthService` from `packages/shared/`. The popup (`App.tsx`) renders 4 view states: `loading`, `login`, `mfa` (static "use web app" message), `authenticated`. Token store uses in-memory cache + `chrome.storage.local`. ADR 0001 documents the service-worker-as-API-proxy pattern.

## What to build

Add registration, MFA challenge, password reset link, and comprehensive error states.

**Registration:** Popup has a login/register toggle. Registration collects name, email, password (client-side validation: 8+ chars, uppercase, digit — matching `PROJECT.password` rules in shared config). On success, auto-logs in (token pair stored, authenticated view shown). On validation error (400/409), shows field-level messages from `AuthValidationError.fields`.

**MFA challenge:** Replace the static "use web app" message with a TOTP input view. When login returns `mfa_pending`, show a 6-digit code input. Submitting calls `POST /api/v1/mfa/validate?code=<totp>` with `Authorization: Bearer <mfa_pending_token>`. Returns a `TokenPair` on success (stored, authenticated view shown). The MFA validate endpoint uses a query param (`code`), not a JSON body — this differs from other endpoints.

**Password reset:** Login form has a "Forgot password?" link that opens the web app's reset page in a new tab. No in-extension reset.

**Error states:**
- Network error (`AuthNetworkError`, fetch throws) — banner with retry button at top of popup
- Invalid credentials (401, `AuthSessionError`) — inline message below form
- Account inactive (depends on API error code — e.g. `ACCOUNT_INACTIVE`) — message + link to support
- Email already registered (409 from register, `AuthValidationError`) — inline field-level error
- Invalid TOTP — inline error on MFA challenge view
- MFA already handled via mfa_pending flow above

**Service worker:** Add `REGISTER`, `MFA_VALIDATE`, `FORGOT_PASSWORD`, `RESET_PASSWORD` message handlers using `AuthService` methods. For `MFA_VALIDATE`, either extend `AuthService` with an `mfaValidate()` method (sending query param + Bearer token) or call `fetch` directly in the handler.

**Types:** Extend `PopupMessage` union in `types.ts`. Extend `PopupResponse` to cover new response shapes.

**Env:** Add `VITE_FRONTEND_URL` (default `http://localhost:3000`) to `.env.example` for the "Forgot password?" link target.
